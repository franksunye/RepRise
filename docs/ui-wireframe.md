# UI 文字版界面（Wireframe-level）与页面结构说明

基于 “管家（线下防水维修销售）＋培训 +持续辅导 +模拟 +业务流程 (电话 → 上门 → 报价 → 成交)” 的业务模型。

下面是系统的核心界面 / 页面（每个页面为一个模块或视图），以及每个页面关键 UI 区块（component）的文字说明，供设计师做原型或 mockup 使用。

---

## 1. 主仪表盘 (Dashboard / Home)

布局 & 关键区块：

- 顶部导航栏 (Top Nav)
  - Logo / 系统名
  - 模块入口（示例：管家首页 / 练习 / 通话 / 任务 / 内容 / 分析）
  - 用户头像 + 下拉菜单（切换视角：管家 / 教练 / 管理端）

- 侧边导航 (Sidebar)
  - 模块项（Onboarding、模拟练习、真实通话、辅导任务、内容库、分析报表）
  - 可展开子模块（例如模拟练习下有电话练习、报价练习）

- 中央主要区域 (Main Content Panel)
  - 今日任务（Today’s Tasks）：显示当天/本周待完成的练习/任务
  - 快速入口（Quick Start）：例如“开始电话模拟练习”、“开始报价练习”、“查看 Playbook”
  - KPI 概览卡（小型卡片）：例如 “练习次数 (本周)”、”通话得分平均值”、”上门谈判任务完成率”

- 页脚（Footer）
  - 帮助 / 文档链接
  - 反馈入口
  - 系统版本信息

---

## 2. 模拟练习 (Role-Play)

布局 & 关键区块：

- 练习类型选择面板
  - 电话练习（Cold call / 跟进）
  - 上门对话练习（勘查场景）
  - 报价谈判练习
  - 异议处理练习

- 练习详情 / 启动区
  - 练习说明（场景简介）
  - “开始练习”按钮
  - 可选：查看脚本 / 提示（话术、要点）

- 对话 / 模拟界面
  - 对话窗口（气泡式）展示 AI 客户与用户对话（文本/语音）
  - 用户输入区（文本 + 麦克风按钮）
  - 对话控制（开始/暂停/结束/重置）

- 练习反馈面板（练习结束后）
  - 多维度评分（专业性 / 话术质量 / 时间管理 / 异议处理）
  - 系统 + 教练 / 同伴文字反馈
  - 反思日志（用户可写练习心得）

- 练习历史记录
  - 列表视图（日期 / 类型 / 得分）
  - 快捷操作（重新练习 / 复习脚本）

---

## 3. 教练仪表盘 (Coach Dashboard)

布局 & 关键区块：

- 教练首页（Summary）
  - 管家列表（姓名 + 练习状态 + 弱项概览）
  - 待办提醒（需要布置或评价的任务）
  - 最近反馈摘要

- 管家详情视图（点击某个管家后）
  - 练习记录（历史练习、得分、反馈）
  - 真实通话分析（集成录音/转录时展示关键对话指标）
  - 辅导计划 / 任务列表（当前布置与完成状态）
  - 行为 & KPI 图表（勘查率、报价转化率、成交率 vs 练习/辅导频率）
  - 教练输入区（添加反馈、设定下次练习任务）

---

## 4. 内容库 / Playbook

布局 & 关键区块：

- 内容中心（Content Hub）
  - 搜索栏 + 过滤（类型、更新日期、标签）
  - 内容卡片（标题、摘要、最后更新时间）

- 内容详情页
  - 标题 + 版本号
  - 正文 / 脚本（支持表格、示例模板、附件）
  - 下载 / 导出（PDF/打印）
  - 版本历史（修改记录、作者、时间）
  - 内容反馈区（用户/教练可提交建议）

---

## 5. 行为 & 分析（Analytics）

布局 & 关键区块：

- 总览（Analytics Home）
  - KPI 卡片 + 图表（练习参与率、通话平均得分、onboarding 完成率）
  - 行为→业务关联视图（练习次数 vs 勘查率、得分 vs 成交率）

- 管家绩效分析
  - 列表 + 搜索：按管家展示关键指标
  - 详情视图含趋势图与教练互动统计

- 教练 ROI 分析
  - 教练投入（任务、反馈） vs 团队绩效提升
  - 不同辅导频率与效果对比图

---

## 6. 任务 & 通知（Tasks & Notifications）

布局 & 关键区块：

- 我的任务（Rep / Coach）
  - 列表：任务类型、截止时间、状态
  - 每条任务可直接“开始任务”进入练习或反思页
  - 通知中心：任务提醒、教练评价提醒

---

## UI 设计建议（通用原则）

- 模块化 & 卡片化：使用卡片展示任务、练习、内容，提升可扫描性
- 仪表板优先：KPI 卡片 + 图表 + 待办提示应布局在首页主要区域
- 简洁导航：顶部 + 侧边导航组合，保持主要功能可达性
- 强搜索/过滤：内容库与练习类型页应支持强搜索与筛选
- 响应式设计：移动端界面要简洁，优先任务与练习入口
- 清晰反馈：练习结果（分数、建议、反思）要突出并易于复盘
- 版本治理：内容条目要展示版本历史与编辑记录
- 任务引导：任务被创建时显示目的、预估时间、期望目标以提高完成率
