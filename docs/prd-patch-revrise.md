基于现有的 PRD +你希望加入「营收赋能 (对话分析 +业绩 +奖励)」的新维度，这里给出 **简洁明了** 的产品设计 (Spec) 修改建议。

---

## 要修改 /补充到 PRD (Spec) 的关键内容

1. **新增模块设计**

   * 对话分析 (Conversation Intelligence)：说明录音、转录、行为评分 (behavioral scoring) 的指标 (如提问比例、异议处理、客户参与度等)。
   * 业绩 (Performance / Revenue) 模块：定义从 CRM 获取商机 / pipeline /签单数据，并展示在管家 /教练 /管理层仪表盘。
   * 奖励和激励机制：设计如何根据练习行为 (role-play、练习次数)、对话得分 + 真实业绩 (签单、勘查成功等) 发放奖励 (积分、徽章、排行榜、现金或非现金)。

2. **调整用户故事**

   * 为 **管家 (Rep)** 加入 “我希望通话分析反馈 +我的真实签单数据能在一个面板里看见，这样我知道练习是否真正转化为业务成果”。
   * 为 **教练 /管家经理** 加入 “我希望能看到教练任务 +练习表现 +真实签单业绩之间的关系，以便辅导时重点关注行为对结果的影响”。
   * 为 **运营 / Enablement 团队** 加入 “我希望分析哪些赋能活动 (练习 /教练) 与业务成果 (签单、勘查率) 强相关，以优化资源投入和内容设计”。

3. **功能需求补充**

   * **对话分析功能**：添加存储 /管理通话录音 + 转录 + 分析模块；提供评分和行为维度。
   * **业绩数据接入**：定义与 CRM 集成 (或通过 API 拉取 pipeline 和签单数据)，以及管家业绩仪表盘 (个人 &汇总)。
   * **奖励机制**：任务 /练习 /对话 /业绩四维触发逻辑 (什么时候发奖励)，排行榜 /徽章 /积分系统。
   * **关联分析报表**：行为 (练习、对话) 与业务 (签单、勘查率) 的关联分析 (ATA，attribution)。
   * **通知机制**：当管家因为某些行为 (练习 /对话得分 /签单) 达到奖励标准时，发通知。

4. **非功能需求补充**

   * **数据安全**：对话录音、行为评分、业绩数据都是敏感，需加密存储 /访问权限控制。
   * **权限设计**：新增角色或权限 (例如 “绩效查看者 /激励管理员”)，明确谁能看对话分析 /业绩 /奖励数据。
   * **性能 /可扩展性**：对话分析模块可能产生大量数据 (录音、转录)，系统要能扩展、快速查询。

5. **验收标准 (Acceptance Criteria) 调整**

   * 验收 “通话分析”：管家完成通话录音 &转录后，系统可以展示行为评分 +分析报告。
   * 验收 “业绩模块”：拉取管家在 CRM 中的签单 /pipeline 数据，并在管家界面 +教练界面展示。
   * 验收 “奖励机制”：当管家满足某行为 + 业绩条件，系统自动给出奖励 (积分 /徽章) 并显示在排行榜 /通知里。
   * 验收 “分析报表”：Enablement 团队可以看到行为 (练习 /对话) 与业务 (签单) 之间的关联图表 /仪表盘。

6. **里程碑 /时间线调整**

   * 在 roadmap 中加上：**对话分析 +录音模块** 的开发与上线 (最好放在第一版或第二版早期)，这样行为数据可以从早期就被捕获。
   * 加入 **奖励机制** 的开发时间 (设计规则、测试、上线)。
   * 加入 **归因分析报表** (行为 vs 业绩) 的设计 +实现里程碑。

7. **风险 & 缓解补充**

   * 风险：对话评分 /分析可能不准确或被管家质疑 — 缓解：先以部分管家试点 +反馈机制优化评分模型。
   * 风险：奖励设计不公平或成本过高 — 缓解：先做试点 (小范围) 验证激励效果 + ROI，然后再大规模推广。
   * 风险：业务 (签单) 数据与行为数据不同步 — 缓解：建立数据同步机制与校验流程。
