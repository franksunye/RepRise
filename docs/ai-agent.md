# AI Agent 层整合设计说明

下面是整合 **AI Agent 能力** 到我们之前设计 (原则 / UI 流程图 /PRD) 的一个 **AI 层 (Agent 层)** 版本说明。这样设计师 /开发团队能更清晰地看到 Agent 在整个系统中的作用，以及它们在 UI /流程中的落点。

---

## 1. 总体架构（AI 层）

* 在之前系统的模块 (模拟练习 /教练 /内容 /分析 /任务) 之上，增加一个 **AI Agent 层**，由多个不同职责 (persona) 的 Agent 组成。
* 每个 Agent 专注一种智能能力 (对话模拟 /内容推荐 /教练 /分析 /后续回顾)，与 UI 模块协作，为用户 (管家 /教练 /运营) 提供智能辅助。

---

## 2. 将 AI Agent 整合到之前的设计原则中

| 原则 | Agent 如何增强 /实现该原则 |
| --- | --- |
| **以结果为导向 (Outcome-driven)** | **Analytics Agent** 分析练习行为与业务 KPI (如勘查率、报价转化率) 的关联，给出改进洞察，帮助团队聚焦对业务绩效真正有影响的训练点。 |
| **角色 & 阶段专门化 (Role & Stage Specificity)** | **Role-Play Agent** 可以根据不同销售阶段 (电话勘查、报价谈判) 和客户 "画像 (persona)" 扮演不同买家，模拟真实对话情景。 <br>**Coaching Agent** 根据管家在练习中的弱项 (异议处理、开场、价值表达) 提建议。 |
| **嵌入工作流 (Flow-of-Work)** | **Content / Copilot Agent** 嵌入在内容界面 /任务界面，为管家提供 "下一步最佳练习 /内容推荐"。 <br> **Follow-Up Agent** 在练习完成后生成复盘与行动任务 (反思 +下次练习)，推动落地。 |
| **实践优先 (Practice)** | **Role-Play Agent** 提供模拟练习环境 (AI 客户对话)；练习后立即给出评分 +反馈。 <br> **Coaching Agent** 建议教练分配具体练习任务，形成练习 +辅导闭环。 |
| **持续教练 (Sustained Coaching)** | **Coaching Agent** 为教练提供智能建议 (任务 +反馈重点)； **Follow-Up Agent** 帮助管家和教练创建复盘与下一步行动计划。 |
| **内容管理 (Content Governance)** | **Content / Copilot Agent** 可基于管家行为推荐最相关 playbook (话术脚本、报价模板)；也可辅助创建内容 (例如帮助生成报价脚本、模拟场景) – 类似 Seismic 的 AI 生成内容能力。 |
| **分析与评估 (Analytics)** | **Analytics Agent** 提供洞察、分析报告、趋势预测；帮助运营 /教练决策优化训练流程，并把行为与业务 KPI 连接起来。 |

## 3. AI Agent 在 UI 流程图 /界面流中的落点 (整合)

下面是一个简化流程 + UI 层级结构 (之前 UI 流图 + Agent 层整合) 的文字说明：

```
[主仪表板 UI]  
   ├── "AI 聊天 /建议" Widget → 启动 **Chat / Copilot Agent**  
   ├── 今日任务 /练习入口  
   └── KPI 卡片 (练习次数 /得分 /业务指标)

[模拟练习页面]  
   ├── 对话区 (Role-Play Agent 扮演买家)  
   ├── 用户输入 (文本 /语音)  
   └── 练习结束 → Role-Play Agent 给出 "评分 +反馈" + "反思日志任务"

[教练仪表盘页面]  
   ├── 管家列表 +练习记录  
   ├── 教练任务管理区  
   └── **Coaching Agent 建议面板**：根据练习历史 +管家弱点推荐任务 +练习

[内容推荐 / Copilot 页面]  
   ├── 输入 /对话框 (管家 /教练提问)  
   └── **Content Agent 推荐内容卡片** (脚本、模板、练习场景) + "布置任务"按钮

[分析 & 报表页面]  
   ├── 图表 (练习数据 /得分 /KPI 关联)  
   └── **Analytics Agent 面板**：洞察 +建议 ("哪些练习对业绩影响最大"、"哪个管家下阶段练习重点")

[练习回顾 /后续页面]  
   └── 练习结束后弹出 **Follow-up Agent 面板**：总结 (Recap)、行动建议 (下一练习)、反思日志 +可 "发送给教练"
```

## 4. PRD (产品需求) 中 AI 层功能补充

在之前 PRD 的功能章节里，新增以下 AI 能力 (Agent 层) 项：

### Role-Play Agent 功能

* 支持生成多种模拟练习场景 (客户 persona、对话阶段)
* 对话评分 +反馈 (速度、异议处理、语气等)
* 练习结束后，生成反思任务 (Reflection)

### Coaching Agent 功能

* 分析练习历史 +角色扮演得分，给教练推荐辅导任务
* 帮助教练设定下次练习重点 /反馈内容
* 支持教练和管家之间任务创建 (教练直接通过 Agent 建议创建任务)

### Content / Copilot Agent 功能

* 根据管家练习表现 (弱点)、deal 阶段推荐 playbook 内容 /话术 /练习场景 (类似 Seismic Aura Chat)
* 支持自然语言查询 (用户可以用对话方式问 "我需要哪个报价脚本？")
* 自动创建练习任务 (通过 "布置练习"按钮)

### Analytics Agent 功能

* 综合分析练习数据、教练互动、业务 KPI 并生成 "洞察 +建议"
* 支持自然语言报表查询 (教练 /运营可以问 "哪些练习最有助于提升成交率？")
* 提供趋势预测 (未来练习重点 /弱项)

### Follow-up Agent 功能 (可选)

* 在模拟练习结束或真实对话结束后自动生成复盘总结 (AI 生成 Recap) (Seismic 的 Aura Follow-up Agent)
* 生成下一步行动 (下一个练习、反思日志)
* 支持把复盘与建议发送给教练 (任务通知)

## 5. 风险 & 注意事项 (Agent 层相关)

### 上下文准确性
Agent (特别是 Role-Play 和 Content Agent) 需要非常贴合你们业务 (管家 +防水维修) 的上下文 (客户谈话、报价、异议)，否则反馈 /推荐可能不准确。

### 数据隐私与安全
AI Agent (分析 +对话) 会处理练习对话、评分、教练反馈等敏感数据，需要有数据治理、访问权限机制。

### 采纳 (Adoption)
管家 /教练是否愿意与 AI Agent 互动 (练习、复盘、任务) 还需设计引导机制 (任务提醒、奖励机制)

### 模型训练 & 维护
Agent（特别是 Role-Play Agent）可能需要训练 (或者根据你们真实对话数据微调)，维护成本不可忽视。

---

## 总结

* 我们在之前设计 (UI／流程／PRD) 中加入了一个 **独立的 AI 层 (Agent 层)**，由多个 Agent 分角色 (Role-Play、Coaching、Content、Analytics、Follow-up) 组成。
* 每个 Agent 都与 UI 模块 (练习页、教练页、内容页、分析页) 对接，为管家 /教练 /运营等角色提供智能辅助。
* 在 PRD 中明确了这些 Agent 的功能：模拟对话 +评分、教练建议 +任务推荐、内容推荐 +任务创建、行为数据分析 +洞察、复盘 +行动建议。
* 同时识别了这个层设计可能带来的风险 (上下文匹配、数据安全、采纳和模型维护)。
